# TTN Implementation Plan
**4 สัปดาห์ - CP352005 Networks**

## สัปดาห์ที่ 1: วางแผน (ToT Phase)
**เป้าหมาย:** ตกลงให้ได้ว่าระบบจะทำอะไรบ้าง

| วัน | กิจกรรม | ของส่ง |
|-----|---------|--------|
| 1 | ตั้งค่า, แบ่งงาน | Repo พร้อมใช้ |
| 2-3 | แต่ละคนเสนอไอเดีย (ToT 5 แขนง) | ตัดสินใจเลือก Multi-Sensory + AI Buffering |
| 4-5 | ออกแบบสถาปัตยกรรมร่วมกัน | architecture_spec.md ฉบับร่าง |

## สัปดาห์ที่ 2: เขียนโค้ด (CoT/ReAct Phase)
**เป้าหมาย:** ให้แต่ละ layer ทำงานได้

| วัน | ใครทำ | อะไร |
|-----|-------|------|
| 6 | Engineer | สร้าง BSAP (ที่อยู่อุปกรณ์) |
| 7 | Engineer | สร้าง TSP (จัดการ session) |
| 8-9 | Engineer + Specialist | สร้าง TRP (หาเส้นทาง) + ทดสอบ latency |
| 10 | Specialist + Engineer | สร้าง NAP + ทดสอบ |

## สัปดาห์ที่ 3: ต่อวงจร (ReAct/RAG Phase)
**เป้าหมาย:** เอาทุกอย่างมาต่อกันให้ได้ ทดสอบความทนทาน และปรับปรุงแก้ไข

| วัน | กิจกรรม |
|-----|---------|
| 11 | เอา BSAP + TSP + TRP ต่อกัน |
| 12 | เอา NAP ต่อ เพิ่มความปลอดภัย |
| 13 | ทดสอบ scenarios 5 แบบ (latency, packet loss, ฯลฯ) |
| 14-15 | ซ้อมนำเสนอภายใน |

## สัปดาห์ที่ 4: ส่งงาน
**เป้าหมาย:** ทุกอย่างพร้อมส่ง

| วัน | กิจกรรม |
|-----|---------|
| 16 | เขียนเอกสาร (ทุกคนช่วยกัน) |
| 17 | - |
| 18 | ทำสไลด์ + ซ้อม |
| 19 | ซ้อมใหญ่ |
| 20 | ส่งงาน |
